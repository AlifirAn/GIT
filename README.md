Как работать с Git\
git init - сделать папку Git-репозиторием(инициализировать)\
git clone <url> (копирование удалённого репозитория)\
git status (проверить состояние)\
git add <путь/файл> (добавить файл к индексу)\
git add . (добавить все изменённые файлы)\
git commit -m "Сообщение коммита" (зафиксировать с сообщением)\
git log (история коммитов)\
git branch (список ветвей)\
git branch <branch-name> (создать новую ветку)\
git checkout <branch-name> (переключиться на ветку)\
git merge <branch-name> (слияние ветки в текущую)
git remote add origin <url> (добавить удалённый репозиторий)\
git remote -v (проверить удалённые)\
git push origin <branch> (отправить ветку на сервер)\
git pull origin <branch> (получить и слить изменения)\

hash коммита - основной идентификатор коммита, позволяет узнать его автора, дату и содержимое закоммиченных файлов, видно в git log(Все хеши, а также таблицу соответствий хеш → информация о коммите Git хранит в папке .git)\
git log (команда для просмотра коммитов репозитория, состоит из хеша коммита; Author; Date; сообщение коммита)\
git log --oneline (команда сокращенного лога коммитов - только первые несколько символов хеша каждого коммита и их комментарии)\
